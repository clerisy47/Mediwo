## Clinical Intake & Patient Information Workflow

### 1. Start Intake Session
**Endpoint**: `POST /api/intake/start`

**Response**:
```json
{
  "sessionId": "session123...",
  "assistantMessage": "Hello! I'm here to help collect your medical history..."
}
```

### 2. Patient Conversations (Multiple Messages)
**Endpoint**: `POST /api/intake/message`

**Request**:
```json
{
  "sessionId": "session123...",
  "message": "I have been experiencing chest pain for 2 weeks"
}
```

**Response**:
```json
{
  "assistantMessage": "I understand. Can you describe the nature of the chest pain?"
}
```

### 3. Complete Intake & Save to MongoDB
**Endpoint**: `POST /api/intake/complete`

**Request**:
```json
{
  "sessionId": "session123...",
  "patientId": "a1b2c3d4e5f6...",
  "doctorId": "x1y2z3a4b5c6..."
}
```

**Response**:
```json
{
  "success": true,
  "summary": "Patient reports 2 weeks of intermittent chest pain. Pain is dull, worse with exertion..."
}
```

**What happens**: 
- Conversation summary is generated by AI
- Medical info is saved to MongoDB `patient_info` collection
- Status is set to "pending" (doctor hasn't reviewed yet)
- Session data is cleaned up

### 4. Save Additional Medical Reports Summary
**Endpoint**: `POST /api/patient-medical-info`

**Request**:
```json
{
  "patient_id": "a1b2c3d4e5f6...",
  "doctor_id": "x1y2z3a4b5c6...",
  "medical_reports_summary": "EKG shows normal sinus rhythm. Troponin levels normal. Chest X-ray clear.",
  "conversation_summary": "Patient reports 2 weeks of intermittent chest pain..."
}
```

**Response**:
```json
{
  "success": true,
  "info": {
    "id": "info_123...",
    "patient_id": "a1b2c3d4e5f6...",
    "doctor_id": "x1y2z3a4b5c6...",
    "medical_reports_summary": "EKG shows normal sinus rhythm. Troponin levels normal. Chest X-ray clear.",
    "conversation_summary": "Patient reports 2 weeks of intermittent chest pain...",
    "status": "pending",
    "created_at": "2025-03-29T12:30:45.123Z"
  }
}
```

## Data Flow Summary

1. **Patient Registration** → Stored in `users` collection
2. **Clinical Intake** → Conversation happens via AI assistant
3. **Intake Complete** → Conversation summary + medical reports saved to `patient_info` collection
4. **Status**: `pending` (waiting for doctor's review)
5. **Doctor Review** → Doctor can see all patients via `/api/doctor/{doctor_id}/patients-ready`
6. **Doctor Adds Notes** → Status changes to `reviewed`, notes stored in `patient_info`
7. **Patient Can View History** → Via `/api/patient/{patient_id}/medical-history`

## Database Schema

### Users Collection
```javascript
{
  "_id": ObjectId,
  "id": String,              // Unique identifier
  "username": String,        // Unique
  "password": String,        // Plain text (development only)
  "full_name": String,
  "role": String,            // "patient" or "doctor"
  "specialization": String,  // Only for doctors
  "available": Boolean,      // For doctors
  "created_at": ISODate,
  "updated_at": ISODate
}
```

### Patient Info Collection
```javascript
{
  "_id": ObjectId,
  "id": String,                      // Unique identifier
  "patient_id": String,              // Reference to users collection
  "doctor_id": String,               // Reference to users collection
  "medical_reports_summary": String, // Summary of medical documents
  "conversation_summary": String,    // Summary of AI conversation
  "doctor_notes": String,            // Optional notes from doctor
  "status": String,                  // "pending", "reviewed", etc.
  "is_ready_for_doctor": Boolean,    // Flag for doctor visibility
  "created_at": ISODate,
  "updated_at": ISODate
}
```

## Troubleshooting

### MongoDB Connection Issues
- Verify MONGODB_URL in .env is correct
- Check network connectivity to cluster0.87bdjhd.mongodb.net
- Ensure credentials are correct (clerisy47:nenOA2qIDG0wD9uM)

### Import Errors
- Ensure pymongo is installed: `pip install pymongo`
- Check that database.py is in the backend/ directory
- Verify sys.path is set correctly in service files

### Service Not Starting
- Check logs for specific error messages
- Verify all dependencies are installed: `pip install -r requirements.txt`
- Test MongoDB connection independently: `python -c "from database import get_database; get_database()"`
